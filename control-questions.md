# Ответы на контрольные вопросы

## Раздел 1: ОС macOS и архитектура

1. **Какая текущая версия macOS?**
   На текущий момент (весна 2026 года) актуальной версией является **macOS 16**.

2. **На основе какой UNIX-подобной системы построена операционная система macOS?**
   macOS построена на базе **Darwin**, которая в свою очередь основана на **FreeBSD** и микроядре **Mach**.

3. **К какому типу относится ядро macOS? В чем отличия между XNU и Mach?**
   * Ядро macOS (XNU) является **гибридным**.
   * **Mach** — это микроядро, которое отвечает за базовые задачи: управление памятью, потоками и передачу сообщений (IPC).
   * **XNU** (X is Not Unix) — это комбинация Mach и основных компонентов FreeBSD. Главное отличие в том, что XNU включает в себя монолитные компоненты (стек протоколов, файловые системы) прямо в пространстве ядра для повышения производительности, чего нет в «чистом» микроядре Mach.

4. **Какой тип архитектуры у процессоров линейки M1 и выше?**
   Процессоры Apple Silicon (M1, M2, M3 и т.д.) используют архитектуру **ARM** (AArch64).

5. **Что такое виртуализация? Приведите примеры гипервизоров.**
   Виртуализация — это технология, позволяющая запускать несколько изолированных операционных систем на одном физическом устройстве.
   * **Примеры гипервизоров:** Parallels Desktop, VMware Fusion, VirtualBox, QEMU.

## Раздел 2: Командная оболочка и терминал

6. **Что такое «командная оболочка» / «командный интерпретатор»?**
   Это текстовый интерфейс (программа), который принимает команды от пользователя, обрабатывает их и передает операционной системе для выполнения.

7. **Как называется командный интерпретатор и какая его версия в используемой ОС?**
   В macOS по умолчанию используется **zsh** (Z shell). Узнать версию можно командой `zsh --version`.

8. **Для каких задач предназначено приложение Terminal в macOS?**
   Для прямого взаимодействия с системой через командную строку: управление файлами, запуск скриптов, настройка системы, работа с сетью и средствами разработки.

9. **Путь к домашней папке и команда для его определения:**
   * Полный путь обычно выглядит так: `/Users/mikhail`
   * Команды: `pwd` (текущий каталог) или `echo $HOME` (путь к домашнему каталогу).

10. **Какой командой можем вывести содержимое каталога системных конфигурационных файлов?**
    ```bash
    ls /etc
    ```

11. **Синтаксис для установки Midnight Commander (mc) в macOS (3 варианта):**
    1. `brew install mc` (через Homebrew)
    2. `sudo port install mc` (через MacPorts)
    3. Сборка из исходников: `./configure && make && sudo make install`

12. **Как вывести текст руководства (справочной информации)?**
    ```bash
    man <имя_команды>
    ```

13. **Для каких целей используется команда open?**
    Для открытия файлов, директорий или приложений из терминала так, как если бы вы использовали Finder (например, `open .` откроет текущую папку в окне).

14. **Примеры команд группировки и перенаправления в zsh:**
    * `>` — перенаправление вывода в файл (перезапись).
    * `>>` — добавление вывода в конец файла.
    * `|` (pipe) — передача вывода одной команды на вход другой.
    * `{ command1; command2; }` — группировка команд.

15. **Команда для изменения командного интерпретатора с zsh на bash:**
    ```bash
    chsh -s /bin/bash
    ```

## Раздел 3: Работа с Git

16. **В каких случаях выполняется инициализация репозитория? Какой командой?**
    Выполняется один раз при создании нового проекта локально командой `git init`.

17. **Что такое коммит?**
    Это фиксированное состояние проекта (снимок), сохраненное в истории репозитория с уникальным хэш-кодом.

18. **Почему перед созданием новой ветки важно создать коммит?**
    Чтобы зафиксировать текущие изменения и иметь «чистую» точку отсчета, исключая перенос незавершенной работы в новую ветку.

19. **Состояния файла в репозитории:**
    * **Untracked** (неотслеживаемый), **Modified** (измененный), **Staged** (подготовленный), **Unmodified** (зафиксированный).
    * Переход: `git add` (в Staged), `git commit` (в Unmodified).

20. **Что такое ветка?**
    Это именованный указатель на определенный коммит, позволяющий вести разработку параллельно основной линии.

21. **Условие для формирования ветки master/main?**
    Необходимо сделать в репозитории хотя бы один (первый) коммит.

22. **Что такое HEAD?**
    Указатель на текущую активную ветку или конкретный коммит, с которым идет работа в данный момент.

23. **Способы создания веток:**
    * `git branch <имя>` (создать)
    * `git checkout -b <имя>` (создать и перейти)

24. **Как узнать текущую ветку?**
    Командой `git branch` (текущая помечена звездочкой).

25. **Как переключаться между ветками?**
    Командой `git checkout <имя_ветки>` или `git switch <имя_ветки>`.

26. **Команда для слияния двух веток:**
    ```bash
    git merge <имя_ветки>
    ```

27. **Как подключить внешний репозиторий?**
    ```bash
    git remote add origin <URL>
    ```

28. **Какая команда позволяет отменить последний коммит?**
    ```bash
    git reset --soft HEAD~1
    ```

29. **Для чего предназначена команда rebase?**
    Для переноса («перебазирования») веток на новый базовый коммит для создания линейной истории.

30. **В каких случаях требуется создание форка? Способы создания:**
    Нужен для работы с чужим проектом, в который нет прав записи.
    * **Способы:** кнопка "Fork" в интерфейсе GitHub или через CLI: `gh repo fork <URL>`.
